# 청담잇츠미 영미권 PR 사이트 — 배포 가이드

이 문서는 사이트를 GitHub에 올리고 Vercel로 배포하는 전체 과정을 단계별로 안내합니다.

---

## 📦 프로젝트 개요

- **기술 스택**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **페이지 6개**: Home / About / Treatments / Results / Pricing / Contact
- **이미지 최적화**: 149MB → 3.2MB (WebP 변환, 97.8% 감소)
- **빌드 결과**: 모든 페이지 정적 생성 (페이지당 ~99kB) — 매우 빠름
- **언어**: 영문 (영미권 글로벌 타겟)
- **가격**: USD 변환 표시 (1 USD ≈ 1,400 KRW 기준)

---

## 🚀 1단계: 로컬에서 사이트 확인 (선택)

zip을 다운받아 압축을 풀고 터미널에서 다음 실행:

```bash
cd itsme-clinic
npm install
npm run dev
```

브라우저에서 `http://localhost:3000` 접속하면 사이트가 보입니다.

> ⚠️ Node.js 18.17 이상 필요 (권장: 20 LTS 또는 22)

## 🐙 2단계: GitHub에 업로드

### 2-1. GitHub 계정/저장소 만들기
1. [github.com](https://github.com) 가입 (이미 있으면 스킵)
2. 우측 상단 **"+" 버튼 → New repository**
3. Repository 이름: `itsme-clinic` (원하는 이름)
4. **Public** 또는 **Private** 선택 (Vercel 무료 플랜은 Private도 가능)
5. **README, .gitignore, license는 추가하지 않기** (이미 프로젝트에 있음)
6. **"Create repository"** 클릭

### 2-2. 로컬 → GitHub 푸시

터미널에서 프로젝트 폴더로 이동 후:

```bash
cd itsme-clinic

# Git 초기화
git init
git add .
git commit -m "Initial commit: IT'S ME Clinic global PR site"

# 메인 브랜치 이름 설정
git branch -M main

# GitHub 저장소 연결 (YOUR_USERNAME을 본인 계정명으로 변경)
git remote add origin https://github.com/YOUR_USERNAME/itsme-clinic.git

# 푸시
git push -u origin main
```

푸시할 때 GitHub 인증 창이 뜨면:
- **Username**: GitHub 아이디
- **Password**: 비밀번호 대신 **Personal Access Token** 입력 필요
  - GitHub에서 Settings → Developer settings → Personal access tokens → Generate new token (classic) → `repo` 권한 체크 → 생성된 토큰 복사

또는 더 쉬운 방법:
- **GitHub Desktop** 앱 사용 (https://desktop.github.com): GUI로 클릭만으로 업로드 가능

## ▲ 3단계: Vercel에 배포

### 3-1. Vercel 가입 + 프로젝트 연결

1. [vercel.com](https://vercel.com) 접속 → **"Sign Up"** → **"Continue with GitHub"** (GitHub 계정 연동)
2. 가입 후 대시보드에서 **"Add New..."** → **"Project"** 클릭
3. 방금 푸시한 `itsme-clinic` 저장소 옆에 **"Import"** 버튼 클릭

### 3-2. 배포 설정

Vercel이 자동으로 **Next.js**를 감지합니다. 다음 설정 그대로 두세요:

| 항목 | 값 |
|---|---|
| Framework Preset | Next.js (자동 감지) |
| Build Command | `npm run build` (기본값) |
| Output Directory | `.next` (기본값) |
| Install Command | `npm install` (기본값) |
| Environment Variables | (추가할 게 없음 — 비워두기) |

**"Deploy"** 버튼 클릭 → 약 2분 후 배포 완료!

배포가 끝나면 자동으로 `https://itsme-clinic-xxxx.vercel.app` 같은 임시 도메인이 생성됩니다.

### 3-3. 자동 재배포

이후로는 GitHub `main` 브랜치에 푸시할 때마다 **Vercel이 자동으로 다시 배포**합니다. 별도 작업 필요 없습니다.

## 🌐 4단계: 도메인 연결 (선택)

### 옵션 A: Vercel 무료 도메인 그대로 사용
- `https://itsme-clinic-xxxx.vercel.app` 그대로 사용 가능
- 추가 비용 없음

### 옵션 B: 본인 도메인 연결 (예: `itsmeclinic.com`)

1. 도메인 구입 (가비아, Namecheap, GoDaddy 등에서 구입)
2. Vercel 프로젝트 → **Settings → Domains**
3. **"Add"** 버튼 → 도메인 입력 (예: `itsmeclinic.com`)
4. Vercel이 보여주는 DNS 레코드를 복사
5. 도메인 등록업체 사이트에서 DNS 설정에 그 레코드 추가
6. 5–30분 후 자동 연결 + SSL 인증서 자동 발급 (HTTPS)

### 옵션 C: 기존 `itsme1.kr` 도메인 사용
- 기존 호스팅과 별개로 운영하려면: 서브도메인 사용 추천 (`global.itsme1.kr`, `en.itsme1.kr`)
- 메인 도메인을 새 사이트로 옮기려면: 기존 호스팅 해지 후 위와 동일한 방법으로 연결

## ✏️ 5단계: 콘텐츠 수정하는 법

### 가장 많이 수정할 곳: `lib/data.ts`

이 파일 하나에 **모든 콘텐츠가 있습니다** — 시술 정보, 가격, 원장님 약력, 클리닉 주소까지.

**예시: 가격 수정**
```typescript
// lib/data.ts 안의 PRICING 객체 수정
ondaTiers: [
  { shots: "40,000 lines", eventPriceKrw: 39, regularPriceKrw: 59 },
  // 39를 다른 숫자로 바꾸면 USD 자동 계산됨
],
```

**예시: 환율 변경**
```typescript
export const USD_RATE = 1400;  // 이 숫자만 바꾸면 모든 USD 가격 자동 업데이트
```

### 수정 후 배포

```bash
git add .
git commit -m "Update pricing"
git push
```

푸시하면 Vercel이 약 2분 안에 자동으로 재배포합니다.

### 가장 쉬운 방법: Formspree (추천)

2. New Form 생성 → form ID 받기 (예: `xnqyldoz`)

   ```tsx
   <form onSubmit={handleSubmit} className="space-y-8">
   ```
   를 다음으로 변경:
   ```tsx
   <form action="https://formspree.io/f/xnqyldoz" method="POST" className="space-y-8">
   ```
4. `handleSubmit` 함수 호출 부분과 submitted 상태 부분 제거 (단순 form 제출로 변경)
5. 푸시하면 끝 — 폼 제출 시 Formspree에서 메일 받음

## 🆘 문제가 생기면

**빌드 실패시**
1. Vercel 대시보드 → 해당 배포 클릭 → **"Build Logs"** 확인
2. 보통 `npm install` 또는 폰트 로딩 문제

**이미지가 안 보일 때**
- 파일명 대소문자 확인 (Mac은 대소문자 무시, Linux는 구분)
- `public/images/...` 경로 확인

**Node.js 버전 문제**
- Vercel은 자동 처리하지만 로컬에서는 18.17 이상 필요
- 확인: `node -v`
